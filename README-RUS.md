# AI Observability Framework — Подробное техническое описание

Этот проект представляет собой высокотехнологичный фреймворк на базе **NestJS**, спроектированный для разработки, мониторинга и оптимизации приложений, использующих большие языковые модели (LLM). NestJS выступает в роли **API-шлюза (API Gateway)**, обеспечивая жесткую типизацию на TypeScript, валидацию данных и отказоустойчивость.

## 🚀 Основные возможности

- **API Gateway**: Принимает входные данные (product_name, category, keywords), валидирует их и управляет лимитами таймаутов (45 секунд).
- **Real-time Streaming**: Возвращает ответ клиенту в реальном времени через Chunked Transfer / Server-Sent Events (SSE).
- **Изолированная AI-логика**: Использование Low-Code оркестратора **Flowise** (внутри Docker на порту 3005) для управления Chatflow на базе LangChain.
- **Advanced Observability**: Полная прозрачность цепочки через **Langfuse Cloud**: трассировочные графы, стоимость токенов, Latency и логирование промптов.
- **Программный Scoring (Quality Gate)**: Автоматическая проверка валидности JSON на стороне бэкенда с проставлением оценки качества (`valid_json: 1/0`) в трейс Langfuse.

## 🛠 Технологический стек

- **Язык и Среда**: Node.js, TypeScript (Строгая типизация, DTO).
- **Бэкенд-фреймворк**: NestJS (Архитектура контроллеров и сервисов, RxJS, управление асинхронными потоками).
- **AI-Оркестрация**: Flowise AI, LangChain.
- **ИИ-Модели**:
    - **Gemini 1.5 Flash**: Основная модель для генерации текста.
    - **text-embedding-004**: Модель для создания семантических векторов.
- **Векторная база данных**: Supabase Cloud, расширение **pgvector** (косинусное сходство, функция `match_documents` на базе PL/pgSQL).
- **AI Observability**: Langfuse Cloud, Langfuse Prompt Registry (версионирование промптов).
- **Сетевой слой**: Axios (потоковое чтение Stream, AbortController для таймаутов).
- **DevOps**: Docker (Multi-stage build), GitHub Actions, GitHub Container Registry (GHCR).

## 🏗 Пайплайн данных

1. **NestJS -> Flowise**: Бэкенд отправляет POST-запрос на Flowise API. Используется динамический ID чатфлоу и `Override Configuration` для проброса переменных в `promptValues`.
2. **Retrieval (Извлечение)**: Входящие данные векторизуются через `text-embedding-004`. Выполняется семантический поиск по базе знаний в Supabase для поиска примеров стиля.
3. **Augmentation & Generation**: Контекст подставляется в шаблон промпта и уходит в **Gemini 1.5 Flash**. Благодаря Structured Output Parser, модель возвращает строго валидный JSON с 5 полями: `title`, `meta_description`, `h1`, `description`, `bullets`.
4. **Monitoring & Scoring**: Каждый шаг логируется в Langfuse. NestJS проверяет JSON и проставляет оценку качества прямо в трейс.

## 🌊 Потоки данных (Streaming & Logic)

В системе реализовано два основных типа обработки данных:

### А. Стриминг через Flowise (External Integration)
Путь запроса: `POST /api/generate-seo`
1. **Controller**: Принимает `product_name`, `category` и `keywords`.
2. **AiService**: Устанавливает HTTP-соединение с внешним сервисом Flowise (`axios` с `responseType: 'stream'`).
3. **Обработка потока**: Сервис в реальном времени читает чанки данных от Flowise.
4. **Трансформация**:
    - Если приходят стандартные SSE-события (`data: ...`), они парсятся.
    - Если прилетает сырой JSON-структурированный текст, он очищается и передается дальше.
5. **SSE (Server-Sent Events)**: Контроллер через `RxJS Observable` транслирует данные клиенту, позволяя видеть процесс генерации SEO в реальном времени.

### Б. Внутренний RAG пайплайн (Internal Processing)
Логика, сосредоточенная в `GenerateSeoUseCase`:
1. **Поиск контекста (Vector Search)**: На основе категории продукта выполняется поиск похожих документов в Supabase (PostgreSQL + pgvector).
2. **Сборка промпта**: Из Langfuse загружается шаблон промпта `seo_description_generator`, в который подставляются найденный контекст и данные продукта.
3. **Генерация**: Запрос отправляется в LLM (Gemini 3.1 Flash Lite) с подключенным обработчиком трейсинга Langfuse.
4. **Валидация и Скоринг**:
    - Ответ проверяется на наличие валидного JSON-блока.
    - В Langfuse автоматически отправляется оценка (score): `1` за успешный парсинг, `0` за ошибку.

## 🧠 Алгоритмы и Инструменты

### Алгоритм RAG (Retrieval-Augmented Generation)
1. **Векторизация**: Использование эмбеддингов для представления знаний в векторном пространстве.
2. **Similarity Search**: Поиск по косинусному сходству в Supabase для извлечения наиболее релевантного контекста.
3. **Augmentation**: Обогащение системного промпта извлеченными данными перед отправкой в LLM.

### Observability (Наблюдаемость)
Интеграция с **Langfuse** позволяет:
- Отслеживать полные трейсы (цепочки вызовов) от входа в Use Case до ответа LLM.
- Видеть время выполнения каждого этапа (Latency).
- Контролировать стоимость (Tokens usage).
- Выполнять автоматический и ручной скоринг ответов.

### Инфраструктурный стек
- **NestJS**: Модульный бэкенд-фреймворк.
- **LangChain**: Оркестрация работы с LLM и промптами.
- **Supabase**: Хранение данных и векторный поиск.
- **Google Gemini**: Основная языковая модель.
- **Axios**: Для устойчивого стриминга данных из внешних API.

## 🛠 Конфигурация и DevOps

Приложение полностью контейнеризировано и интегрировано в современный CI/CD процесс:

- **Docker**: Использование Multi-stage build для минимизации веса образа. Изоляция сети и проброс портов (3005:3000).
- **CI/CD**: Пайплайн в **GitHub Actions**:
    - Автоматическое тестирование и линтинг при пуше.
    - Сборка иммутабельного Docker-образа через Docker Buildx.
    - Публикация в **GitHub Container Registry (GHCR)**.
- **Развертывание**: Готовность к **Zero-Downtime Rolling Update** (например, в Azure Container Apps через Azure Pipelines).
