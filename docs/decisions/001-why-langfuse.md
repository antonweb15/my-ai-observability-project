# ADR 001: Использование Langfuse для управления промптами

## Статус
Принято

## Контекст
Нам необходимо управлять промптами для AI-пайплайна. Варианты включали хранение промптов в коде, в MD-файлах в репозитории или использование внешней системы (Prompt Management System).

## Решение
Выбрано использование **Langfuse Prompts**.

## Обоснование
1.  **Версионирование без деплоя**: Промпты можно обновлять через UI Langfuse, не перезапуская приложение.
2.  **Обсервабильность**: Прямая связь между версией промпта и трейсом выполнения.
3.  **Удобство для не-разработчиков**: Маркетологи или AI-инженеры могут править промпты в UI.

## Последствия
- В MD-файлах в проекте хранятся только описания логики промптов и их связей, но не сам текст промпта.
- Код должен реализовывать интерфейс `IPromptProvider` для абстракции от Langfuse.
