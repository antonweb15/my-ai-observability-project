# Процесс генерации SEO (Workflow)

Этот документ описывает бизнес-логику процесса генерации SEO-описания для товара с использованием RAG (Retrieval-Augmented Generation).

## Схема процесса

1.  **Получение контекста**: Поиск в векторной БД (Supabase) по категории товара.
2.  **Загрузка промпта**: Получение актуального промпта из Langfuse (`seo_description_generator`).
3.  **Генерация**: Вызов LLM (Google AI) с использованием контекста и данных товара.
4.  **Валидация**: Проверка ответа на соответствие формату JSON.
5.  **Скоринг**: Отправка оценки качества (valid_json) в Langfuse.

## Используемые промпты (Langfuse)
- `seo_description_generator`: Основной промпт для генерации. Принимает переменные `productName`, `category`, `context`.

## Правила валидации
Ответ модели должен быть валидным JSON-объектом. В случае ошибки парсинга, в Langfuse отправляется score `value: 0`.
